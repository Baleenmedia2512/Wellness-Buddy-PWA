## [Unreleased] (previous)
### Added
- feat(diary): add trend charts + a working "Other" (unknown) card flow to the single Diary page. (1) `shell/components/DiarySummaryCards.jsx` now renders each feature's real **trend** component under its summary — calorie trend (`NutritionFilters` via `useCalorieTrend`+`useCalorieChartData`), weight trend (`WeightChart` via `useWeightDashboard`), education trend (`EducationTrendPanel`). (2) New `shell/components/UnknownEntryFlow.jsx`: tapping an Other row opens the image with **Retry** (re-runs Gemini → promotes capture `unknown→food`) and **Edit** (pick Food / Weight / Education → transfers the data to that vertical: food promotes the capture, weight saves a weight record, education saves a log). Trend components + their hooks exposed via the feature barrels (VSA §2.2). [ai-assisted: Copilot (Claude Opus 4.8)]
- feat(diary): show the real per-feature summary cards on the single Diary page. New `shell/components/DiarySummaryCards.jsx` re-mounts the actual `NutritionSummaryCards`, `WeightSummaryCards`, and `EducationSummaryCards` components — fed by their existing feature hooks (`useDayAnalyses` + `useUserCalorieTarget` + `useBurnedCalories`, `useWeightHistoryData`, `useEducationDashboard`) — stacked above the diary feed, so the consolidated page keeps the same nutrition (calories + burn-to-balance), weight (current/previous/low/high), and education (streak/week) summaries as the old tabs. Replaces the earlier generic derived strip. Summary cards + their hooks are now exposed via the weight / education / nutrition feature barrels (VSA §2.2). [ai-assisted: Copilot (Claude Opus 4.8)]
- feat(diary): make the Diary the single dashboard surface (ADR-0003 follow-on). When `ff.diary-feed` is ON, `shell/components/Dashboard.js` hides the Food / Weight / Education tab strip and renders only the unified Diary feed with a day stepper (defaults to today). A derived summary strip (calories in / burned / net / latest weight / education count) replaces the separate per-tab summary cards — computed from the day's diary entries, no extra fetches. Tapping a row opens the matching detail card hosted in the shell: weight → existing `WeightCardModal` (view + edit + delete), education/watch → `EducationCardModal`, food → new standalone read-only `nutrition/components/FoodDetailModal.jsx`. Delete / weight-edit reuse the existing per-feature API services and remount the feed. Legacy per-tab dashboards remain mounted only when the flag is OFF. Flag default flipped to ON. [ai-assisted: Copilot (Claude Opus 4.8)]
- chore(test): wire `setupFilesAfterEnv: src/setupTests.js` in `frontend/jest.config.js` so direct `jest` runs load the `@testing-library/jest-dom` matchers (`toBeInTheDocument`, etc.) — previously only `react-scripts test` loaded them, so single-file `jest` runs of the diary suite failed under the standalone runner. New `GET /api/background-analysis/captures/unknown-share` returns the capture image plus a server-computed `canMutate` flag (owner OR a coach in the owner's upline; anonymous link recipients get image-only). New `features/captures/components/UnknownShareViewer.jsx` renders the image card with Retry / Edit actions gated on `canMutate`; new `features/captures/api/captureShareClient.js` (`fetchUnknownShare`, `promoteUnknownToFood`). `App.js` deep-link handler routes an `unknown` share to the viewer when `ff.diary-feed` is ON: Retry re-runs Gemini and promotes `unknown → food`; Edit opens `SmartFoodSearchModal` and promotes with the picked nutrition. Mutation re-checks the permission policy server-side, so a forged `canMutate` cannot promote. No DB migration; additive response fields only. Default OFF via `ff.diary-feed`. Requires `@principal-eng` review (public share surface, §5.5). [ai-assisted: Copilot (Claude Opus 4.8)]
- feat(captures): AI detection failures land as Diary "Other" rows instead of prompting (PR-D of ADR-0003). When `ff.diary-feed` is ON, the `App.js` low-confidence branch no longer opens `UnknownCaptureModal` — the capture stays tagged `unknown` and surfaces as an Other row with Retry / Edit. Flag OFF preserves the legacy PR-3 disambiguation modal. [ai-assisted: Copilot (Claude Opus 4.8)]
- feat(weight): celebration animation on weight loss progress. When users upload weight image and AI detects progress (≥0.1 kg loss vs previous entry), app displays joyful confetti burst **immediately on Home screen** after weight is saved, with personalized message and optional musical chord. New `shared/components/CelebrationConfetti.js` (Canvas + framer-motion), integrated into `App.js` weight upload flow. Celebration triggers in `performWeightSave()` after successful save and weight history comparison. Docs: `frontend/src/features/weight/CELEBRATION.md`. [ai-assisted: Copilot (Claude Sonnet 4.5)]
- feat(diary): new `features/diary/` slice + 4th "Diary" tab in `Dashboard.js` (PR-C of ADR-0003). Cards-only newest-first feed wired to `GET /api/diary/list` (PR-B). Row types: Food / Weight / Education / Watch / Other. Mounted only when the new `frontend/src/config/featureFlags.js :: ff.diary-feed` is enabled — default OFF, env override `REACT_APP_FF_DIARY_FEED=true`, per-device override via `localStorage.setItem('ff.diary-feed','true')`. Owns its own lazy chunk so OFF means zero cost. The 3 existing tabs (Food / Weight / Education) keep working unchanged. Includes new `frontend/src/setupTests.js` (CRA-standard `@testing-library/jest-dom` setup that was missing). [ai-assisted: Copilot (Claude Opus 4.7)]
- refactor(shell): extract `DashboardTabs.jsx` from `Dashboard.js` (PR-C cleanup). Pulls the tab-strip JSX + the two custom inline-SVG icons (WeighingScale / Education) out into a sibling presentational component. Brings `Dashboard.js` back under the §2.3 fail threshold of 500 LOC (was 558, now 442). Behaviour-preserving.
- refactor(shell): move `Dashboard.js` from `shared/components/` to new `shell/components/` composition layer (F1 prerequisite of PR-C, executes [ADR-0001](docs/adr/0001-shell-composition-layer.md) Track A). Eliminates the `shared-cannot-import-features` dep-cruiser violation. New `frontend/src/shell/README.md` documents the layer charter, current resident, and planned residents (Header.js next).
- feat(background-analysis): `GET /api/diary/list` joined feed read-model (PR-B of ADR-0003 Diary consolidation). Returns food + weight + education + watch entries for a `{ ownerUserId, date }` window, sorted newest-first. Inclusion of `unknown` captures is gated by the new `ff.diary-feed` server-side feature flag (default OFF). Reuses the PR-A retry permission policy so coach-of-owner reads are allowed and audit-logged on the same posture as `retryPromotionToFood`. Per-vertical reads degrade independently — one failing stream warn-logs and returns empty without taking the feed down. New `backend/shared/lib/feature-flags.js` registry with stale-flag detector. [ai-assisted: Copilot (Claude Opus 4.7)]
- refactor(background-analysis): extract `diary.service.js` for `resolvePublicCapture` and `retryPromotionToFood` (PR-B prerequisite). No runtime behaviour change. Brings `analysis.service.js` from 595 LOC down to 423; new `diary.service.js` houses the diary-related orchestrators. Tests + route imports updated.
- feat(background-analysis): `POST /api/background-analysis/captures/retry-promotion` endpoint (PR-A.2 of ADR-0003 Diary consolidation). Promotes an `unknown` capture to `food` in place after the user (or a coach in the user's upline) supplies a fresh Gemini analysis (Retry) or a manual nutrition payload (Edit). Reuses `analysis.service.save` for the upsert, the captures state machine for the transition, and the new `captures/domain/permissions/retry.policy.js` for authorisation. Coach actions on member captures are audit-logged. No DB migration; no breaking change. [ai-assisted: Copilot (Claude Opus 4.7)]
- feat(captures): allow `unknown → food` capture state transition (PR-A of ADR-0003 Diary consolidation). Adds pure `domain/permissions/retry.policy.js` deciding whether a viewer is the owner or in the owner's coach upline, ready for PR-A.2 to wire into a `POST /api/captures/retry-promotion` endpoint. Locks in food-corrections search latest-wins dedup via regression test (no runtime behaviour change). [ai-assisted: Copilot (Claude Opus 4.7)]
- feat(quick-share): public recipient-viewable link. `POST /api/quick-share/captures` returns `{ token, viewUrl, expiresAt }`; Gemini food analysis runs in background; `GET /api/quick-share/public/:token` and `/s/[token]` serve a PII-redacted nutrition view. 30-day expiry. Camera-first flag is now **default ON** for members (opt out via `REACT_APP_FF_QUICK_SHARE_CAMERA_FIRST=false`). Migration: `add_quick_share_public_token.sql` (2 nullable columns on `food_nutrition_data_table`). [ai-assisted: Copilot]
- feat(quick-share): camera-first capture with direct WhatsApp image share (flagged off by default behind `REACT_APP_FF_QUICK_SHARE_CAMERA_FIRST`). Member taps shutter → photo shared directly via existing shareImageDirectly → navigates Home. No backend required.
