## [Unreleased]
### Added
- feat(background-analysis): `GET /api/diary/list` joined feed read-model (PR-B of ADR-0003 Diary consolidation). Returns food + weight + education + watch entries for a `{ ownerUserId, date }` window, sorted newest-first. Inclusion of `unknown` captures is gated by the new `ff.diary-feed` server-side feature flag (default OFF). Reuses the PR-A retry permission policy so coach-of-owner reads are allowed and audit-logged on the same posture as `retryPromotionToFood`. Per-vertical reads degrade independently — one failing stream warn-logs and returns empty without taking the feed down. New `backend/shared/lib/feature-flags.js` registry with stale-flag detector. [ai-assisted: Copilot (Claude Opus 4.7)]
- refactor(background-analysis): extract `diary.service.js` for `resolvePublicCapture` and `retryPromotionToFood` (PR-B prerequisite). No runtime behaviour change. Brings `analysis.service.js` from 595 LOC down to 423; new `diary.service.js` houses the diary-related orchestrators. Tests + route imports updated.
- feat(background-analysis): `POST /api/background-analysis/captures/retry-promotion` endpoint (PR-A.2 of ADR-0003 Diary consolidation). Promotes an `unknown` capture to `food` in place after the user (or a coach in the user's upline) supplies a fresh Gemini analysis (Retry) or a manual nutrition payload (Edit). Reuses `analysis.service.save` for the upsert, the captures state machine for the transition, and the new `captures/domain/permissions/retry.policy.js` for authorisation. Coach actions on member captures are audit-logged. No DB migration; no breaking change. [ai-assisted: Copilot (Claude Opus 4.7)]
- feat(captures): allow `unknown → food` capture state transition (PR-A of ADR-0003 Diary consolidation). Adds pure `domain/permissions/retry.policy.js` deciding whether a viewer is the owner or in the owner's coach upline, ready for PR-A.2 to wire into a `POST /api/captures/retry-promotion` endpoint. Locks in food-corrections search latest-wins dedup via regression test (no runtime behaviour change). [ai-assisted: Copilot (Claude Opus 4.7)]
- feat(quick-share): public recipient-viewable link. `POST /api/quick-share/captures` returns `{ token, viewUrl, expiresAt }`; Gemini food analysis runs in background; `GET /api/quick-share/public/:token` and `/s/[token]` serve a PII-redacted nutrition view. 30-day expiry. Camera-first flag is now **default ON** for members (opt out via `REACT_APP_FF_QUICK_SHARE_CAMERA_FIRST=false`). Migration: `add_quick_share_public_token.sql` (2 nullable columns on `food_nutrition_data_table`). [ai-assisted: Copilot]
- feat(quick-share): camera-first capture with direct WhatsApp image share (flagged off by default behind `REACT_APP_FF_QUICK_SHARE_CAMERA_FIRST`). Member taps shutter → photo shared directly via existing shareImageDirectly → navigates Home. No backend required.
- Engineering governance framework: `claude.md`, `/governance/`, PR template, CODEOWNERS, CI workflows, automation scripts.
